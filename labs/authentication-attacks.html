Windows Event ID 4625: Failed Logon Detection

Event ID 4625 is generated whenever a logon attempt fails on a Windows system. SOC analysts monitor this event to detect:

  • Brute force attacks
  • Password spraying attempts
  • Credential stuffing
  • Unauthorized access attempts

Event Fields for Investigation:

Field Description Investigative Value
Account Name Username that failed authentication Identify targeted accounts
Failure Reason Why the login failed Bad password vs. account disabled
Logon Type How authentication was attempted Interactive (2) vs. Network (3)
Source IP Where attempt originated External vs. internal threat
Workstation Name Source computer name Identify attack source

Wazuh SIEM Detection: Rule 60122

Wazuh Rule 60122 triggers on Windows failed logon events, providing centralized monitoring and alerting for authentication failures.

Rule Details:

Rule ID: 60122 Level: 5 (Medium) Description: "Logon failure - Unknown user or bad password" Event ID: 4625 Data Source: Windows Security Log MITRE ATT&CK: T1078 (Valid Accounts)

Why This Matters for SOC Analysts:

  • Centralized visibility across all monitored systems
  • Real-time alerting on authentication failures
  • Pattern recognition for multiple failures
  • Integration with MITRE ATT&CK framework

Account Lockout Detection: Event ID 4740

When multiple failed authentication attempts exceed the lockout threshold, Windows generates Event ID 4740 - indicating a potential brute force attack.

Event ID 4740 Analysis:

Event ID: 4740 Description: "A user account was locked out" Target Account: [Username] Caller Computer: [Source that caused lockout] Timestamp: When lockout occurred
Critical Alert: Event ID 4740 indicates defensive controls activated after multiple failed attempts. This requires immediate SOC investigation to determine if attack is ongoing or if account owner needs assistance.

SOC L1 Investigation Workflow

Step 1: Initial Alert Review

  • ✅ Alert received: Wazuh Rule 60122 (Failed logon)
  • ✅ Severity assessment: Level 5 (Medium)
  • ✅ Event source: Domain Controller (DC01)
  • ✅ Event ID: 4625

Step 2: Evidence Gathering

  • ✅ Identified target account: testuser
  • ✅ Source IP address: [Extracted from event]
  • ✅ Failure reason: Bad password
  • ✅ Timeline: When attempts occurred

Step 3: Pattern Analysis

Questions to answer:

  • How many failed attempts occurred?
  • Were attempts from single source or multiple sources?
  • Is this normal user behavior or potential attack?
  • Did attempts trigger account lockout?

Step 4: MITRE ATT&CK Mapping

Technique ID Evidence
Valid Accounts T1078 Failed authentication attempts
Account Access Removal T1531 Account lockout triggered

Step 5: Classification & Response

L1 Analyst Decision:
• Classification: Potential credential access attempt
• Severity: Medium (no successful compromise detected)
• Response: Monitor for continued attempts
• Escalation: Escalate to L2 if pattern continues
• User notification: Contact account owner to verify activity

Detection Patterns for SOC Analysts

Normal User Behavior:

  • 1-2 failed attempts followed by success
  • Attempts from known IP addresses
  • Consistent timing (during business hours)
  • User reports forgotten password

Suspicious Indicators:

  • Multiple rapid failed attempts
  • Attempts from unusual IP addresses
  • Off-hours authentication attempts
  • Multiple accounts targeted from same source
  • Account lockout triggered
