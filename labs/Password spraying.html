Executive Summary

Attack vs. Brute Force: Understanding the Difference

In my previous lab, I simulated a brute force attack (T1110.001) that targeted one account with multiple passwords - triggering account lockout. Password spraying is more sophisticated:

Aspect Brute Force (T1110.001) Password Spray (T1110.003)
Target 1 user, many passwords Many users, 1 password
Speed Fast (seconds) Slow (hours/days)
Detection Easy - triggers lockout Hard - requires correlation
Real-World Use Less common Very common (APTs, ransomware)

Attack Execution: Kali Linux Simulation

for user in JohnDoe testuser; do smbclient -L //192.168.10.10 -U threatwatch\\$user%Password1234! done

What this does:

  • Targets 2 domain accounts (JohnDoe, testuser)
  • Tries same password (Password1234!) against each
  • Only 1 attempt per account - stays under lockout threshold
  • Uses SMB protocol (common in enterprise lateral movement)
Attack Success: testuser account compromised without triggering any lockout alerts - demonstrating why password spraying is more dangerous than brute force.

Detection: SIEM Analysis

Despite the stealthy nature, the attack generated detectable patterns in Wazuh SIEM:

Key Detection Indicators:

  • Same source IP: 192.168.10.131 (KALI-01)
  • Multiple users targeted: JohnDoe, testuser
  • Event pattern: 4625 (failed) → 4624 (success) from same source
  • Suspicious workstation: KALI-01 (penetration testing OS)
  • NTLM authentication: Legacy protocol, security concern

Forensic Analysis: Event ID 4624 (Successful Logon)

Account Name: testuser Account Domain: THREATWATCH Logon Type: 3 (Network - SMB) Source Workstation: KALI-01 Source IP: 192.168.10.131 Authentication Package: NTLM Elevated Token: Yes (privileged access!)
Red Flags: Linux workstation name (KALI-01) + NTLM + Elevated privileges + Success after failures = High-confidence attack indicator

Forensic Analysis: Event ID 4625 (Failed Logon)

Correlation Analysis:

Field Failed (JohnDoe) Success (testuser)
Source IP 192.168.10.131 192.168.10.131 ✅
Workstation KALI-01 KALI-01 ✅
Target JohnDoe testuser ❌

Pattern: Same source, different targets = Password spraying

Defensive Recommendations

  1. Deploy MFA: Completely mitigates password spraying
  2. Ban common passwords: Password1234!, Welcome123, etc.
  3. Enhanced monitoring: Alert on multiple users from same source IP
  4. Network segmentation: Limit SMB access to authorized systems
  5. Regular password audits: Identify weak credentials before attackers do
