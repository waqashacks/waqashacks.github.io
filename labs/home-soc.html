Infrastructure Deployment

1. pfSense Firewall Configuration

Deployed pfSense as the network gateway to simulate enterprise network segmentation and routing.

Configuration:

  • LAN interface: 192.168.10.1/24
  • DHCP server for automatic IP assignment
  • Firewall rules for internal network traffic
  • Gateway for internet access

2. Active Directory Domain Controller (DC01)

Deployed Windows Server 2022 as domain controller to simulate enterprise authentication infrastructure.

Configuration:

  • Domain: threatwatch.local
  • Created organizational units (OUs) for HR, IT departments
  • User accounts: testuser, JohnDoe, Administrator
  • Group Policy implementation

3. Wazuh SIEM Deployment

Installed Wazuh on Ubuntu Server to provide centralized security monitoring and log analysis.

Deployment Steps:

  • Ubuntu Server 22.04 LTS installation
  • Wazuh manager, indexer, and dashboard deployment
  • SSL certificate configuration
  • Web interface access on port 443

4. Endpoint Agent Deployment

Installed Wazuh agents on Windows endpoints to collect security events and forward to SIEM.

Agent Configuration:

  • Agent ID 004: DC01 (Domain Controller)
  • Agent ID 006: HR-PC01 (Windows 11 endpoint)
  • Real-time log forwarding enabled
  • Windows Security event collection
Result: All agents reporting successfully to Wazuh SIEM with real-time log collection from domain controller and endpoints.

Network Architecture Design

Design Principles:

  • Segmentation: Internal network (192.168.10.0/24) isolated from external threats
  • Centralization: All security logs flow to Wazuh SIEM for correlation
  • Scalability: Architecture supports adding additional endpoints and servers
  • Realism: Mirrors enterprise network topology with firewall, AD, SIEM
